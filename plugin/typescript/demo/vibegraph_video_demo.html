Step 1

Register onchain profiles.

The plugin defines register_profile. Each user signs a transaction that stores handle, bio, score, and activity counters in Canopy plugin state.

Step 2

Alice gives Bob a vibe.

The second custom transaction is give_vibe. It records the sender, receiver, score amount, tag, note, and height as a permanent reputation attestation.

Step 3

Bob's reputation increases.

DeliverTx updates both profiles, writes a new VibeAttestation, and increments the attestation counter. The social graph now has a portable reputation edge.

The app touches the local chain.
